# OPAL RM Frontend ExecPlan Guidance

## Overview
Use this skill only when working with ExecPlans for `opal-rm-frontend`.

## ExecPlans
- Read `references/plans.md` in this skill and follow it exactly.
- Store new ExecPlan files under `.codex-plans/` at the repo root.
- Keep plans aligned with the starter repo shape: Angular SSR shell, dashboard-level routing, shared auth/bootstrap, and the intentionally reduced test surface unless the task explicitly expands beyond that scope.
